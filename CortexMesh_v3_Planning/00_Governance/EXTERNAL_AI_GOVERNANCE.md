# External AI Governance

**Status:** Ratified

**Session:** 09

**Authority:** BD-09-003 Governance Safeguards

---

# Purpose

Define the governance boundaries for the use of external AI systems within CortexMesh.

These provisions ensure that external AI assistance supports the project without altering governance authority or accountability.

---

# Governance Principle

External AI systems are advisory and supportive tools.

Responsibility for implementation remains with the Product Owner.

Governance authority remains exclusively with the Board.

No external AI system may independently authorize, approve, ratify, or modify governance decisions.

---

# External AI Roles

## Implementation Assistant

Implementation assistants may:

* generate source code
* prepare documentation
* suggest implementation approaches
* perform refactoring
* assist with testing

Implementation assistants shall operate only within the scope authorized by the Product Owner and, where applicable, the Board.

Implementation assistants possess no governance authority.

Examples include coding assistants such as Codex, Claude Code, or future implementation-capable AI systems.

---

## Independent Reviewer

Independent reviewers may:

* review architecture
* review implementation
* perform audits
* identify risks
* recommend improvements

Independent reviewers may recommend actions but shall not approve, authorize, or ratify governance decisions.

Examples include external architectural or audit reviews performed by AI systems.

---

## Board Member

Board members perform governance functions under the Communication Protocol.

Only Board members may:

* cast governance votes
* approve governance decisions
* ratify Board motions
* establish governance precedent

---

# Product Owner Responsibilities

The Product Owner shall:

* disclose the use of external AI systems where materially relevant;
* define the intended role of each external AI system before engagement;
* ensure external AI activities remain within their authorized scope;
* present external AI outputs as evidence or recommendations rather than governance decisions.

---

# External AI Register

The following register documents the roles of external AI systems used within CortexMesh.

| AI System                   | Role                       | Governance Authority |
| --------------------------- | -------------------------- | -------------------- |
| Codex                       | Implementation Assistant   | None                 |
| Claude Code (or equivalent) | Implementation Assistant   | None                 |
| Claude                      | Independent Reviewer       | None                 |
| Future external AI systems  | Declared before engagement | None                 |
| ChatGPT                     | Board Member               | Board Authority      |
| Kimi                        | Board Member               | Board Authority      |
| DeepSeek                    | Board Member               | Board Authority      |

---

# Governance Principle

Changing implementation assistants or independent reviewers does not modify the governance structure.

Governance authority is determined by role, not by the identity of the AI system.
