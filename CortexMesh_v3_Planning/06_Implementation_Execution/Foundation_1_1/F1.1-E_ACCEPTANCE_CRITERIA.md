## Mandatory rejection conditions

Implementation acceptance must be withheld if any of these occurs:

- generic Python/platform compatibility remains or is newly implied;
- dependency profiles cannot be reproduced under the verified constraints;
- a credential or local path appears in tracked artifacts;
- state mutation is undisclosed or runtime state behavior changes;
- optional service setup becomes default execution;
- Docker provisioning is invented;
- packaging metadata or distribution behavior appears;
- Action SHAs, pip-audit policy, or CI authority changes;
- any pre-existing regression fails;
- any protected identity changes;
- E1/E2 evidence changes;
- implementation exceeds the accepted manifest.

services, capability-image provisioning, installable packaging, or external
distribution.
