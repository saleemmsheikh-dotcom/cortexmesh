## Purpose

This document inventories repository hygiene issues and recommends a
disposition for each item. It does not authorize or perform any move, rename,
deletion, consolidation, regeneration, or ignore-rule change.

The recommendations below are proposals for a later reviewed execution phase.
Published governance records, certified baselines, research evidence, and
historical traceability take precedence over cosmetic repository cleanup.

## Disposition vocabulary

| Term | Meaning |
| --- | --- |
| Retain | Keep the item in its current role and location. |
| Relocate | Preserve the item but move it to a more appropriate location after reference review. |
| Archive | Preserve as historical material outside the active working surface. |
| Remove | Delete only after ownership, reference, and provenance checks are approved. |
| Regenerate | Recreate from an authoritative source using a documented process. |
| Review | Evidence is insufficient for a safe structural disposition. |

No recommendation in this report is self-executing.

Phase 2 decisions are recorded in
`F1.1-C_REPOSITORY_DISPOSITION.md`. Where a Phase 1 recommendation and a
Phase 2 disposition differ, the accepted Phase 2 disposition controls.

## Executive findings

1. The active runtime and test layout is readily identifiable and should not be
   altered by repository hygiene work.
2. `CortexMesh_v3_Planning/` is the authoritative planning tree. The two
   suffixed planning trees are redundant archive candidates already covered by
   the Repository Lineage investigation; their disposition still requires
   explicit approval.
3. Certified baselines and RP-001 result data are historical or scientific
   evidence, not cleanup targets.
4. Several root-level audit, remediation, generated-report, and visualization
   artifacts would be better grouped under controlled tooling or evidence
   locations.
5. One tracked diagram is zero bytes. Other zero-byte tracked files are either
   valid Python package markers or possible negative/failed-run evidence and
   must not be removed indiscriminately.
6. The existing `.gitignore` covers the generated material currently visible in
   the working copy. A later phase may add common editor, build, type-checker,
   and temporary-file patterns.

## `.gitignore` review

### Existing coverage

The current rules appropriately ignore:

- Python bytecode and `__pycache__`;
- pytest and coverage outputs;
- macOS `.DS_Store`;
- virtual environments and `.env`;
- the memory backup file;
- `*.ext4`, `*.zip`, and `*.bundle`.

### Recommended additions for a later phase

| Pattern | Purpose | Qualification |
| --- | --- | --- |
| `.idea/` | JetBrains IDE state | Add unless shared project configuration is intentionally adopted. |
| `.vscode/` | VS Code workspace state | Add unless selected shared settings are intentionally versioned. |
| `.mypy_cache/` | Type-checker cache | Generated and reproducible. |
| `.ruff_cache/` | Linter cache | Generated and reproducible. |
| `.tox/` and `.nox/` | Test environment state | Generated and reproducible. |
| `build/` and `dist/` | Packaging output | Add when packaging workflows are introduced or confirmed. |
| `coverage.xml` and `.coverage.*` | Alternate coverage outputs | Complements existing `.coverage` and `htmlcov/` rules. |
| `*.tmp`, `*.swp`, and `*~` | Temporary/editor files | Prevent accidental local scratch artifacts. |
| `Thumbs.db` | Windows filesystem metadata | Cross-platform repository hygiene. |
| `.env.*` with an explicit `!.env.example` exception | Local secrets/configuration variants | Apply only with a documented example-file convention. |

A broad `*.log` rule is not recommended. CortexMesh deliberately versions
some raw logs and research evidence; a global rule could silently hide required
artifacts. Any log exclusion should be path-scoped and tied to an output policy.

1. remove only ignored local caches from the working copy;

3. resolve the empty diagram and generated root call-graph artifacts;

Each group should be independently reviewable. No architectural consolidation
should be combined with these hygiene actions.
