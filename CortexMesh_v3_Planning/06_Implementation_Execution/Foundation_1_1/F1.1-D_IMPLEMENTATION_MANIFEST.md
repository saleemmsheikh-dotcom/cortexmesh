## Protected-path policy specification

IM003 uses one pattern per non-comment line. IM004 interprets patterns using
documented repository-relative prefix or glob semantics. Invalid patterns fail
closed.

The policy detects and blocks for review. It does not define authority,
supersede the Lock Registry, or permit an automated bypass.

## IM004 behavior contract

The checker must:

- accept an explicit base and head Git revision;
- obtain changed paths from Git using no shell interpolation;
- normalize repository-relative POSIX paths;
- reject absolute paths and parent traversal;
- ignore blank lines and comments;
- support documented exact, directory-prefix, and limited glob patterns;
- sort and deduplicate findings deterministically;
- exit zero when no protected path changed;
- exit nonzero when a protected path changed or policy input is invalid;
- print each matched path and pattern;
- perform no write, network, provider, runtime, or research action; and
- expose importable pure functions for focused testing.

## Verification sequence

1. Run focused IM006 tests.
2. Run canonical `unittest discover`.
3. Run `pytest` and compare expanded test count.
4. Compile IM004, IM006, and active Python scope without repository bytecode.
5. Run positive and negative protected-path fixtures.
6. Run `git diff --check`.
7. Compare the branch diff with IM001–IM010.
8. Verify P0/LOCKED identities unchanged.
9. Push branch and open a pull request.
10. Inspect first workflow runs without retry or gate weakening.
11. Stop and preserve evidence on unexpected behavior.
12. Create IM007 only from factual successful or failed run evidence.
13. Obtain Product Owner acceptance before merge.
14. Do not change SET001–SET006 without separate authorization.

## Rollback

- Before merge, preserve logs and abandon or revert scoped branch commits.
- After merge, use a normal reviewed revert; do not rewrite history.
- Workflow rollback must not erase GitHub run or pull-request evidence.
- Restore any separately authorized repository setting from its recorded prior
  value.
- Re-run local regression and protected-identity checks after rollback.

## Stop conditions

Stop immediately if:

- an Action SHA differs from the manifest;
- Python 3.14 or current requirements cannot install on the selected runner;
- baseline or expanded test counts differ unexpectedly;
- a workflow requires a secret or write permission;
- a protected or unlisted path must change;
- an artifact would contain prohibited data;
- the dependency audit cannot distinguish tool failure from findings;
- GitHub interprets triggers or permissions differently from the accepted
  design;
- required failures are masked; or
- any SET action becomes necessary before separate authorization.
