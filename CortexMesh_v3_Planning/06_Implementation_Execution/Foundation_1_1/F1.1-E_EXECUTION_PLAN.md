## Objective

Plan a controlled implementation of the minimum repository-first environment
contract. This plan does not authorize execution.

## Required precondition

Prepare and obtain Product Owner acceptance of a versioned
`F1.1-E_IMPLEMENTATION_MANIFEST.md` before creating an implementation branch or
editing any candidate implementation file.

The manifest is required because:

- the candidate surface contains more than documentation;
- dependency declarations and an exact constraints record affect environment
  resolution;
- two established CI workflows require narrow command changes;
- the plan must preserve immutable Action SHAs;
- a stateful runtime must be documented without being changed;
- negative tests are needed to prevent packaging and secret leakage;
- protected identities and E1/E2 evidence hashes must be preserved.

## Contract-test design

The future `tests/test_f11e_environment_contract.py` should verify:

1. README links to `ENVIRONMENT.md`;
2. support statement names CPython 3.14 and Ubuntu CI;
3. generic compatibility claims are absent;
4. default verified install command uses the constraints file;
5. canonical unittest command is unchanged;
6. optional OpenAI file is separate and referenced accurately;
7. Local AI variables match current code names;
8. Docker is not presented as a Local AI prerequisite;
9. `cortexmesh-capability` is marked operator/unprovisioned;
10. `memory/memory.json` mutation is explicitly disclosed before CLI use;
11. no reset/delete/state-relocation instruction exists;
12. repository root is the supported working directory;
13. requirements and constraints files are internally consistent;
14. exact constraints contain no editable/local path or URL credential;
15. workflow installs use approved constraints;
16. workflow Action SHAs remain the F1.1-D pins;
17. workflows remain read-only and do not use secrets;
18. no `pyproject.toml`, `setup.py`, `setup.cfg`, or distribution workflow is
    introduced;
19. E1/E2 hashes remain unchanged;
20. optional profiles do not change default runtime behavior.

Tests must inspect behavior/contracts without importing or invoking external
services.

## Stop conditions

Stop immediately if:

- a candidate file conflicts with the manifest;
- dependency resolution requires a source/runtime change;
- an Action SHA or pip-audit pin would change;
- a required dependency cannot resolve on verified CPython 3.14 Ubuntu;
- optional OpenAI installation requires credentialed execution;
- clean-clone verification mutates tracked state;
- an import, memory, provider, research, validation, governance, or LOCKED file
  would need modification;
- packaging metadata or a Docker image appears necessary;
- generated constraints contain local paths, credentials, or unreviewed
  indexes;
- existing tests regress;
- scope exceeds the accepted manifest.

Preserve evidence and request review; do not improvise.

## Rollback

- Before merge: preserve logs and abandon or normally revert scoped branch
  commits.
- After merge: use reviewed revert commits; never rewrite published history.
- Restore previous requirements and workflow install commands together to
  avoid declaration/enforcement mismatch.
- Remove only artifacts explicitly created by the reverted implementation.
- Re-run baseline dependency installation, 250-test regression, and protected
  identity checks.
- Do not erase CI, pull-request, or dependency-resolution evidence.
