## Rollback

1. Stop at the first failed action or verification.
2. Preserve terminal output and observed state without claiming success.
3. Do not proceed to later execution groups.
4. Before merge, abandon the dedicated branch if restoration is required.
5. After a committed group, use a normal Git revert; do not rewrite history.
6. Verify restored P1 files against the hashes recorded in this manifest.
7. Re-run the full regression and P0 integrity checks.
8. Record the failure and disposition before proposing another attempt.

## Acceptance gate for merge

Merge to `main` is prohibited until:

- all in-scope action IDs are PASS or explicitly removed by a new accepted
  manifest version;
- all deferred and prohibited items remain untouched;
- each execution group has an independently reviewable commit;
- structural, hash, regression, whitespace, and boundary checks pass;
- the branch diff contains no unlisted cleanup;
- review is complete; and
- merge is explicitly authorized.
