PASS. There is no composite score, compensating control, majority decision,
confidence adjustment, or waiver implied by a strong result elsewhere.

## Explicit rejection conditions

Implementation must be rejected or returned for remediation if it:

- changes executable application behavior;
- changes or invokes Local AI/provider behavior;
- edits runtime, research evidence, validation certification, governance
  authority, or LOCKED surfaces;
- silently reduces or skips tests;
- treats coverage, latency, or audit counts as a compensating score;
- grants write permission without separately accepted necessity;
- requires production/provider secrets;
- uploads protected or sensitive evidence;
- permits required failures to pass;
- modifies dependencies automatically;
- publishes a package or deployment;
- enables GitHub settings outside recorded authorization;
- contains actions absent from the accepted implementation manifest; or
- cannot be rolled back without history rewriting.

processes. It does not certify production readiness, security, universal Python
compatibility, research validity beyond existing certification, or authority to
change protected components.
